PICPLANR VERSION 37.4 — 40 DAY MEDIA RETENTION

Upload the ZIP contents into the root of the current PicPlanr GitHub repository.

Replace:
1. index.html
2. package.json
3. vercel.json

Add:
4. api/media/register.js
5. api/cron/cleanup-media.js
6. supabase/media-retention.sql

SUPABASE SETUP
1. Open Supabase SQL Editor.
2. Run supabase/media-retention.sql.
3. Create a private Storage bucket named:
   picplanr-media

VERCEL SETUP
Add a new environment variable:
CRON_SECRET

Use a long random value with at least 16 characters.
Apply it to Production and Preview.

RETENTION RULES
Original video footage: 40 days
Temporary exports: 7 days
Captions, schedules, thumbnails and post history remain saved.

IMPORTANT
The cleanup system deletes files that have been registered in media_assets.
The existing browser uploader still analyses files locally. When permanent
cloud video upload is connected, it must call api/media/register.js after a
successful Supabase Storage upload.

The cleanup job runs daily at 04:15 UTC.
