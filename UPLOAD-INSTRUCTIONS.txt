PICPLANR VERSION 38.1 — SECURE SOCIAL OWNERSHIP

NUMBER 4 FOUNDATION
Connected Instagram, LinkedIn and TikTok accounts belong to the correct
authenticated user and workspace.

WHAT IS SECURED
• Every social connection stores workspace_id and user_id.
• A verified provider account ID is required.
• Access and refresh tokens are encrypted with TOKEN_ENCRYPTION_KEY.
• Customers cannot create fake connection rows from the browser.
• Connection status is loaded through an authenticated server route.
• Disconnect requests are limited to the signed-in owner and workspace.
• OAuth state values expire after 10 minutes and are single-use.
• The server ignores browser-supplied user_id and workspace_id values.

IMPORTANT
Instagram, LinkedIn and TikTok are not advertised as live by this version.
Their OAuth authorisation URLs and callbacks will be installed only after
each platform application is approved.

SETUP
1. Upload all ZIP contents to the GitHub repository.
2. Run supabase/secure-social-ownership.sql in Supabase SQL Editor.
3. Redeploy PicPlanr.
4. Log out and log back in.

EXPECTED SQL RESULT
Success. No rows returned
