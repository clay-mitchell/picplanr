PICPLANR VERSION 41.5 — INSTAGRAM LOGIN SESSION DOMAIN FIX

Replace:

index.html
api/oauth/instagram/start.js
api/oauth/instagram/callback.js

CAUSE

The current front end redirects www.picplanrapp.com to picplanrapp.com, while the
Meta redirect URI is configured on www.picplanrapp.com. Supabase browser sessions
are stored per web origin, so moving between www and the apex domain can make the
customer appear logged out after Instagram returns.

FIX

• www.picplanrapp.com is now the single canonical domain.
• picplanrapp.com and Vercel preview domains redirect to www.picplanrapp.com.
• Instagram OAuth state and fallback URLs use the same www origin.
• The popup returns to the already logged-in PicPlanr window instead of opening a
  fresh login session on another domain.

VERIFY THESE SETTINGS

Vercel:
PICPLANR_APP_URL=https://www.picplanrapp.com

Meta redirect URI:
https://www.picplanrapp.com/api/oauth/instagram/callback

Supabase Site URL:
https://www.picplanrapp.com

Supabase Redirect URLs should include:
https://www.picplanrapp.com/**

After deployment, clear Safari website data for both picplanrapp.com and
www.picplanrapp.com once, then log in again on www.picplanrapp.com before testing.
