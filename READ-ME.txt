PICPLANR VERSION 41.7 — SAFARI INSTAGRAM POPUP FIX

Replace only:

index.html

ROOT CAUSE

Safari was blocking the Instagram popup because PicPlanr attempted to open it
after waiting for the OAuth start request. When that happened, PicPlanr fell
back to sending the main app tab to Meta. Returning from Meta then loaded the
login screen.

FIX

• PicPlanr now opens a blank connection popup immediately when the user clicks.
• The authenticated OAuth request runs after the popup already exists.
• The popup is then redirected to Meta.
• The main PicPlanr tab never leaves the app.
• If Safari blocks the popup, PicPlanr shows a clear message instead of
  navigating the logged-in app away.

TEST

1. Deploy index.html.
2. In Safari, allow popups for www.picplanrapp.com if prompted.
3. Log into PicPlanr.
4. Open Connected accounts.
5. Click Connect Instagram.
6. Complete Meta authorisation in the popup.
7. The original PicPlanr tab should remain logged in and show Instagram as
   connected.

No Meta, Supabase, Stripe or Vercel setting changes are required.
