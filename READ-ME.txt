PICPLANR VERSION 40.4 — IMAGE USAGE TRACKING FIX

Replace only the root index.html file in GitHub.

CAUSE
The main PicPlanr image analysis request was using a normal browser fetch.
It was not sending the signed-in Supabase access token to /api/picplanr.
The billing service therefore could not reliably attach image usage to the
correct workspace.

FIX
• All PicPlanr artificial intelligence requests now use authenticatedFetch.
• Image analysis usage is attached to the signed-in workspace.
• The subscription usage display refreshes automatically after an image
  analysis or caption regeneration.

IMPORTANT
The image already analysed before this fix will not be added retrospectively.
After deployment, analyse one new image. The counter should move to 1 of 250.

No Stripe, Supabase or Vercel settings need to be changed.
